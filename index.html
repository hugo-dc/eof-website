Introduction to EVM Object Format

EVM Object Format (EOF) introduces a currently not existent structure to EVM bytecode, defining a clear separation of sections (i.e. code, data, etc). It also introduces a way to create new versions of EOF code, without the risks of breaking backwards compatibility, as each EOF version is independent of one another. Another improvement is that EOF code is validated at deploy time, and not every time the EVM bytecode needs to be executed.
